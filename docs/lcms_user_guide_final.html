1. System Overview

This section provides a high-level description of what the LCMS accomplishes and why it is essential for your organisation’s legal operations.

Key Terms: LCMS – Legal Case Management System; KPI – Key Performance Indicator.

The LCMS centralises case records, timelines, documents, and approvals ensuring transparency and accountability throughout the legal lifecycle.

2. Access & User Roles

Understand what each role can see and do within LCMS so that responsibilities are clearly divided.

Key Terms: RBAC – Role-Based Access Control; CRUD – Create, Read, Update, Delete.
RoleMain ActionsRestrictions
AdminManage users, settings, backupsNone
SupervisorApprove execution, monitor workloadNo system settings
LawyerCreate & update assigned casesCannot view other lawyers’ notes
ClerkLog filings & feesNo approvals

3. Getting Started

This part guides first-time and returning users through access and navigation basics.

3.1 Logging In

  1. Browse to the LCMS URL.
  2. Enter your credentials and click Login.
  3. Change password on first login.

3.2 Navigating the Interface

  • Sidebar – module links

4. Managing Cases

Learn how to create, edit, and maintain case files over their lifecycle.

Key Fields: Status – current stage; File # – internal reference.

4.1 Creating a Case

  1. Click Cases ▸ New.
  2. Select type.
  3. Fill mandatory fields.
  4. Click Save.

4.2 Updating a Case

  • Add documents via drag-and-drop.
  • Post progress updates.
